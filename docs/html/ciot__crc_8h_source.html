21/* Notes about CRC APIs usage
22 * The ESP32 ROM include some CRC tables and CRC APIs to speed up CRC calculation.
23 * The CRC APIs include CRC8, CRC16, CRC32 algorithms for both little endian and big endian modes.
24 * Here are the polynomials for the algorithms:
25 * CRC-8 x8+x2+x1+1 0x07
26 * CRC16-CCITT x16+x12+x5+1 0x1021
27 * CRC32 x32+x26+x23+x22+x16+x12+x11+x10+x8+x7+x5+x4+x2+x1+1 0x04c11db7
28 *
29 * These group of CRC APIs are designed to calculate the data in buffers either continuous or not.
30 * To make it easy, we had added a `~` at the beginning and the end of the functions.
31 * To calculate non-continuous buffers, we can write the code like this:
32 * init = ~init;
33 * crc = crc32_le(init, buf0, length0);
34 * crc = crc32_le(crc, buf1, length1);
35 * crc = ~crc;
36 *
37 * However, it is not easy to select which API to use and give the correct parameters.
38 * A specific CRC algorithm will include this parameters: width, polynomials, init, refin, refout, xorout
39 * refin and refout show the endian of the algorithm:
40 * if both of them are true, please use the little endian API.
41 * if both of them are false, please use the big endian API.
42 * xorout is the value which you need to be xored to the raw result.
43 * However, these group of APIs need one '~' before and after the APIs.
44 *
45 * Here are some examples for CRC16:
46 * CRC-16/CCITT, poly = 0x1021, init = 0x0000, refin = true, refout = true, xorout = 0x0000
47 * crc = ~crc16_le((uint16_t)~0x0000, buf, length);
48 *
49 * CRC-16/CCITT-FALSE, poly = 0x1021, init = 0xffff, refin = false, refout = false, xorout = 0x0000
50 * crc = ~crc16_be((uint16_t)~0xffff, buf, length);
51 *
52 * CRC-16/X25, poly = 0x1021, init = 0xffff, refin = true, refout = true, xorout = 0xffff
53 * crc = (~crc16_le((uint16_t)~(0xffff), buf, length))^0xffff;
54 *
55 * CRC-16/XMODEM, poly= 0x1021, init = 0x0000, refin = false, refout = false, xorout = 0x0000
56 * crc = ~crc16_be((uint16_t)~0x0000, buf, length);
57 *
58 *
59 */
